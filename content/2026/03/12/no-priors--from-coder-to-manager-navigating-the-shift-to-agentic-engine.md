Notion is transforming from a human-centric productivity tool into a platform where humans manage autonomous AI agents that can write their own code and build custom integrations. Co-founder Simon Last reveals that Notion's internal development now relies heavily on coding agents, with the company experiencing a fundamental shift from "humans doing work with tools" to "humans managing swarms of agents."

---

- **Autonomous agent capabilities** Notion's AI agents go beyond using existing tools—they can autonomously build their own integrations and write code to complete tasks, representing a leap beyond traditional AI assistants

- **Internal transformation** Notion itself now uses coding agents extensively in its development process, with teams shifting from direct coding to managing and directing AI agents that write the actual code

- **Data integration challenge** The company has tackled complex semantic indexing across disparate sources like Slack, Google Drive, and other enterprise tools to enable agents to work with real business context

- **Platform strategy pivot** Notion positions itself as "Switzerland" for AI models, allowing customers to use various AI providers while Notion handles the integration complexity and agent orchestration

- **API design for agents** The company is redesigning its APIs specifically for agent customers rather than human users, recognizing that agents have different interaction patterns and capabilities than humans

- **Management paradigm shift** Simon describes a fundamental transition in how work gets done—from humans using tools directly to humans managing and directing autonomous agents that handle execution

- **Rapid iteration cycle** Notion operates on six-month rewrite cycles to keep pace with the fast-evolving AI landscape, indicating the speed required to stay competitive in agentic AI
