Anthropic's Claude Cowork emerged from an unexpected discovery: users were leveraging Claude Code for messy knowledge work rather than just programming tasks. Felix Rieseberg and his team capitalized on this by creating a VM-based desktop environment that gives Claude its own computer, enabling trusted task execution without constant human approval while maintaining security through sandboxing.

---

- **Accidental product discovery** - The Cowork team noticed Claude Code users doing expense management, file organization, and other non-technical work, revealing demand for agentic workflows beyond coding
- **10-day development cycle** - Built by combining existing internal prototypes and leveraging Claude Code as the foundation, demonstrating how cheap execution enables "build all candidates" product development
- **VM-powered architecture** - Claude runs in a lightweight virtual machine that can install tools and execute scripts independently while maintaining safety boundaries through network controls and sandboxing
- **Local-first strategy** - Anthropic is betting on desktop integration over cloud-only solutions, allowing users to work with local files and existing workflows rather than rebuilding everything from scratch
- **Skills-based extensibility** - Simple markdown instructions serve as a lightweight abstraction for reusable workflows, creating portable automation that could transfer between AI products
- **Chrome integration advantage** - Direct browser control eliminates the complexity of managing multiple MCP connectors while providing powerful web automation capabilities
- **AI vertical compression thesis** - Better general models and stronger primitives may absorb many specialized AI wrappers, with the real value shifting to platforms that combine multiple capabilities
- **Autonomous work delegation** - The ultimate goal is moving users from question-answering to trusting AI with longer-duration, higher-stakes tasks that require minimal human intervention
